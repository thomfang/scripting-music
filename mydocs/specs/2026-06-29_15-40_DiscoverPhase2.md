# Spec — 发现页 Phase2：新歌榜 + 为你推荐（种子艺人）

- 状态：APPROVED（决策：①新歌作独立项不分流派；②为你推荐置顶）
- 深度：standard（在已稳定的发现页上加 2 个数据能力 + UI 区块）
- 关联：`2026-06-29_14-55_DiscoverTab.md`（发现 Tab 基础）

## 1. 核心目标
在「发现」页补两块内容：
1. **🆕 新歌榜**：欧美各 genre 的最新发行歌曲（带 30s 试听）。
2. **为你推荐（Phase2）**：基于种子艺人 Radiohead / Novo Amor / Cigarettes After Sex 拉同艺人热门曲，做一个推荐区。

## 2. 接口事实（已 curl 验证）
- **旧版 iTunes RSS 无歌曲级「新歌」feed**：`topsongs`(热门)/`topalbums` 有效；`newreleases`/`newmusic` 400/404；新版 marketingtools `new-releases` 也 404。
- **结论：新歌用 iTunes Search API + 客户端按 `releaseDate` 降序**。
  - `https://itunes.apple.com/search?term=<genre词>&media=music&entity=song&country=us&limit=N`（**必带 UA**，gzip）。
  - Search/Lookup 的 track 字段齐全：`previewUrl`(30s 直链)、`artworkUrl100`(升600)、`trackName`/`artistName`/`collectionName`、`trackTimeMillis`、`releaseDate`、`trackId`。→ 完全可复用 `itunes_preview` 播放链路。
- **为你推荐基础**：`https://itunes.apple.com/lookup?id=<artistId>&entity=song&limit=N`（首条是 artist，其余是 track，含 previewUrl/封面）。
  - 种子 artistId：Radiohead=657515；Novo Amor / Cigarettes After Sex 需运行时用 Search `entity=musicArtist` 解析一次（或硬编码已知 id）。

## 3. 设计

### 3.1 charts.ts 扩展
- 新增 `ChartSource` 概念：`type ChartKind = "top" | "new"`。
  - `top`：现有 RSS topsongs（热门）。
  - `new`：Search API 按 genre 关键词取 → 按 releaseDate 降序 → 映射 ChartTrack。
- 新增 `fetchNewSongs(genreLabel, limit=40)`：Search → 过滤近 N 天/直接按 releaseDate 排序 → ChartTrack[]（previewUrl 必有，缺则丢）。
- 新增 `fetchArtistTop(artistName, limit=12)`：Search `entity=musicArtist` 得 artistId → lookup 拉热门曲 → ChartTrack[]。
- 复用现有 `parseEntry` 思路写 `parseSearchTrack(r)`（JSON 字段而非 RSS）。

### 3.2 发现页 UI
- **决策①**：「🆕 新歌」作为流派条里的一个**独立特殊项**（CHART_GENRES 加 `{id:-1,key:"new",label:"新歌",emoji:"🆕"}`，综合最新、不分流派）。选中它 → 走 `fetchNewSongs`（多 genre 关键词混合 Search + releaseDate 降序）。其余流派项不变走 RSS topsongs。
- **决策②**：「为你推荐」横向卡片墙置于发现页**最顶部**（流派 chips 之上）。`ScrollView axes=horizontal` 大封面卡片，点卡=试听并入队推荐列表。种子艺人常量，解析失败则整区静默隐藏。

### 3.3 播放
- 全部复用 `trackToPreviewMusic` + `player.setQueue/play`（与现有一致，同一区块内连续试听）。

## 4. Done Contract
- `scripting-ts run` 实跑 `fetchNewSongs` 与 `fetchArtistTop` 各返回 ≥1 条且含 previewUrl。
- 发现页能切「热门/新歌」并渲染；推荐区横向滚动显示种子艺人曲。
- preview_ui 渲染通过，无运行时错误。
- 真机：点新歌/推荐曲出声、连续试听 —— 用户确认。
- 回写 Change Log / Validation。

## 5. 风险
- Search「新歌」靠 term=genre 词，精度不如官方榜；用 releaseDate 排序 + genre 关键词缓解。
- 种子艺人 id 解析失败 → 推荐区降级隐藏，不阻塞主榜。
- 推荐区 + 双榜会增加首屏请求；用现有 10min 缓存 + 失败静默。
