# Spec — 发现 Tab（另类/独立榜单）+ 上次 Tab 恢复

- 状态：DONE（preview_ui 验证通过；真机交互待用户确认）
- 深度：deep（新增首页 Tab + 新数据源 + 多文件 + 播放链路复用）
- 关联：`2026-06-29_12-55_MP3JuiceSource.md`（搜索/播放/下载已有链路）

## 1. 核心目标（Loop Anchor）

在 Scripting Music 首页新增「发现」Tab：拉取**美区 iTunes RSS 榜单**，按用户口味（Radiohead / Novo Amor / Cigarettes After Sex 这类**欧美另类/独立**）做**流派分类切换**；列表点歌 = **30s 官方 preview 即时试听**（零解析），需要完整播放/下载时复用现有 mp3juice 链路（歌名+艺人）。**不再用国区 most-played**。
附带：App 每次启动恢复**上次停留的 Tab**。

## 2. 接口事实（已 curl 验证）

- 榜单端点：`https://itunes.apple.com/us/rss/topsongs/limit=<N>/genre=<id>/json`（免 key；`/us/` 美区）。
- entry 字段：`im:name.label`(歌名)、`im:artist.label`(艺人)、`im:collection.im:name.label`(专辑)、`im:image[]`(多尺寸封面，取最大再升级 600x600)、`id.attributes.im:id`(trackId)、`link[]` 里 `rel=enclosure` 的 `href` = **30s preview m4a 直链**（`audio-ssl.itunes.apple.com/...mzaf_*.m4a`，Apple 官方 CDN）、`im:duration.label`=30000。
- 单条 entry 时 `feed.entry` 是对象，多条时是数组 → 需归一化。
- genre id：Alternative=20、Singer/Songwriter=10、Electronic=7、Rock=21、Pop=14、Hip-Hop/Rap=18。（默认顺序见 §4）

## 3. 设计

### 3.1 新数据源 `class/sources/charts.ts`
- `type ChartGenre = { id: number; key: string; label: string; emoji?: string }`，导出 `CHART_GENRES`（口味预设清单）。
- `type ChartTrack = { id; trackId; title; artist; album; cover; previewUrl; duration; provider:"itunes_preview" }`。
- `async fetchChart(genreId: number, limit=40, country="us"): Promise<ChartTrack[]>`：fetch RSS（带 UA）→ 解析归一化 → cover 升级 600x600。
- 失败抛错；调用方兜底。可加内存缓存（genreId→{data,ts}，TTL 10min）。

### 3.2 播放（两级）
- **试听**：构造 `Music`：`provider:"itunes_preview"`、`audio_url=previewUrl`、`is_downloaded:false`、`duration:30`、`source_id=trackId`。直接 `player.play(music)` → 命中 player.ts 现有「非 mp3juice + audio_url 有值 → 直接播」分支，**player.ts 零改动**。
- **完整播放/下载**：用 `${title} ${artist}` 调 `music.search()`（mp3juice）取首条 → 走既有 resolve/下载。放在歌曲行的菜单项「完整播放」「下载」。

### 3.3 发现页 `page/discover/`
- `index.tsx`：顶部横向流派分类（Picker 或横向 ScrollView chips）+ 榜单 List。
- 复用 `SongRow`/卡片样式；行点击=试听，行内菜单=加入播放列表/完整播放/下载。
- loading / error / 空态。
- 「为你推荐」区（可选，Phase2）：种子艺人 Radiohead/Novo Amor/Cigarettes After Sex，用 iTunes lookup 拉同艺人热门曲。本期先做榜单分类，推荐留 Phase2。

### 3.4 首页集成 + Tab 持久化 `page/index.tsx` & `index.tsx`
- 新增 `<Tab title="发现" systemImage="sparkles" value={2}>`（注意 value 2 当前空缺；setting 占 3）。
- `selection` 初始值改为从 `Storage.get("home_selected_tab")` 读，缺省回退资料库(1)。
- `selection` 变化时 `Storage.set("home_selected_tab", v)` 持久化。

## 6. 风险
- iTunes RSS 偶发限流/字段缺失 → 解析做防御（缺 enclosure 的 entry 丢弃或标记不可试听）。
- preview 仅 30s：UI 需标注「试听 30s」，完整播放走 mp3juice。
- TabView selection 用 number；Storage 存取注意类型。
